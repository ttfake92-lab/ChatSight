# ChatSight 产品设计文档

## 1. 项目概述

### 1.1 产品定位

ChatSight 是一款基于 **wechat-cli** 的 AI 驱动微信群消息智能分析与监控桌面端产品。通过本地读取微信数据，结合大模型能力，为用户提供零封号风险的社群运营解决方案。

### 1.2 核心价值

- **绝对安全**：纯只读模式，不涉及任何发消息、修改等高危操作，100% 零封号风险
- **隐私保护**：聊天记录索引与检索全程在本地完成，数据不出机器
- **AI 赋能**：接入主流大模型，一键生成群日报、提取核心观点

### 1.3 目标用户

- **社群运营**：需要管理多个活跃群，每天需要整理群日报、提取用户反馈
- **行业观察者**：加入了大量行业交流群，希望通过关键词监控过滤噪音
- **团队管理者**：需要沉淀内部沟通群的知识，方便后续检索

### 2.3 无数据库设计

**为什么不需要数据库？**

1. **数据来源是只读的**：wechat-cli 本身已是数据源，消息记录直接从微信读取
2. **AI 摘要不需要持久化**：可以实时生成，保存用文件存储（JSON/Markdown）
3. **配置数据很简单**：AI API Key、关键词配置等存放在 `config.json` 或 `.env`

**什么时候需要数据库？**

- 需要存储大量历史消息用于长期分析
- 需要复杂查询（多条件筛选、时间范围查询）
- 需要用户数据（多用户、权限管理）
- 需要离线缓存（减少 wechat-cli 调用）

**初期决策**：不需要数据库，用文件存储即可。如果后续需要，再考虑引入 SQLite 或 IndexedDB。

## 3. 功能模块设计

### 3.1 会话列表模块

#### 功能描述
展示最近会话，支持搜索和筛选

#### API 调用
```bash
wechat-cli sessions --limit 20
```

#### 数据结构
```typescript
interface Session {
  id: string;              // 会话 ID
  name: string;            // 会话名称
  type: 'private' | 'group'; // 会话类型
  lastMessage: string;     // 最后一条消息
  lastMessageTime: string; // 最后消息时间
  unreadCount?: number;   // 未读消息数
}
```

#### 状态管理
使用 Zustand store `useSessionStore` 管理会话列表状态

### 3.2 聊天记录模块

#### 功能描述
展示选中会话的消息历史，支持分页加载

#### API 调用
```bash
wechat-cli history "会话名" --limit 50
```

#### 数据结构
```typescript
interface Message {
  id: string;              // 消息 ID
  sender: string;          // 发送者名称
  content: string;         // 消息内容
  timestamp: string;      // 消息时间戳
  isSelf: boolean;        // 是否为自己发送
}
```

#### 用户交互
1. 点击会话列表中的会话项
2. 加载该会话的历史消息
3. 支持滚动加载更多历史消息

### 3.3 AI 摘要模块

#### 功能描述
对选中会话生成 AI 摘要，提取核心观点和重要信息

#### 工作流程
1. 获取最近 N 条消息（默认 50 条）
2. 调用 AI API（支持 OpenAI GPT / Claude / 本地模型）
3. 展示结构化摘要结果

#### AI 提示词模板

```markdown
## 角色设定
你是一位专业的微信群运营分析师，擅长从聊天记录中提取有价值的信息。

## 分析维度

请对以下微信群聊天记录进行全面分析，输出结构化的分析报告：

### 1. 📊 今日概览
- 群内总消息数量
- 参与讨论的人数
- 讨论的活跃时段
- 与昨日相比的变化趋势

### 2. 🔥 核心话题
- 排名前 3 的热门话题
- 每个话题的讨论热度（高/中/低）
- 话题之间的关联性

### 3. 💡 重要观点
- 用户提出的关键建议或反馈
- 达成的共识
- 存在的分歧或争议点

### 4. ❓ 常见问题（FAQ）
- 用户提问频率最高的问题
- 对应的解答或解决方案

### 5. ⚠️ 待跟进事项
- 需要群主/管理员回复的问题
- 需要后续处理的事项
- 涉及的重要决策点

### 6. 📈 趋势洞察
- 用户关注点的变化趋势
- 潜在的机会或风险提示

## 输出要求
- 使用中文输出
- 结构清晰，分点明确
- 重点信息用 emoji 标注
- 控制总字数在 500 字以内

## 聊天记录内容：
{messages}

## 输出格式：
请按以下 JSON 格式输出：
{
  "overview": { ... },
  "topics": [...],
  "keyPoints": [...],
  "faqs": [...],
  "actionItems": [...],
  "insights": [...]
}
```

#### AI 模型支持
- **OpenAI GPT 系列**：生态最完善，推荐作为默认选项
- **Claude（Anthropic）**：中文理解能力强，适合复杂分析
- **本地模型**：完全隐私，但配置复杂

### 3.4 监控告警模块（第二阶段）

#### 功能描述
实时监控群消息，关键词触发桌面通知

#### API 调用
```bash
wechat-cli new-messages  # 获取增量新消息
```

#### 关键词监控
- 支持多关键词配置
- 支持正则表达式
- 支持关键词组合（AND/OR）

#### 告警机制
- 桌面原生通知（使用 Electron Notification API）
- 支持声音提醒
- 支持静默时段配置

### 3.5 数据分析模块（第三阶段）

#### 功能描述
多维度数据看板，助力社群精细化运营

#### API 调用
```bash
wechat-cli stats  # 聊天统计分析
```

#### 数据指标
- 消息量趋势（近 7 天 / 30 天）
- 活跃时段分布
- 发言排行 Top N
- 消息类型分布（文字/图片/链接等）

### 3.6 搜索与 Skill 系统（第四阶段）

#### 全文搜索
```bash
wechat-cli search "关键词" --chat "会话名"
```

#### Skill 系统
通过 YAML 配置监控规则和 AI 分析动作：

```yaml
name: 竞品监控
description: 监控竞品相关讨论
triggers:
  - keywords: ["竞品", "对比", "替代"]
  - keywords: ["A产品", "B产品"]
actions:
  - type: notify
    title: "发现竞品讨论"
  - type: ai_summary
    template: "竞品分析"
```

## 5. 错误处理机制

### 5.1 wechat-cli 执行错误

```typescript
type WeChatError = 
  | { code: 'NOT_INITIALIZED'; message: '请先运行 wechat-cli init' }
  | { code: 'NOT_FOUND'; message: '未找到该会话' }
  | { code: 'EXECUTION_FAILED'; message: string }
  | { code: 'TIMEOUT'; message: '执行超时' };
```

### 5.2 AI API 调用错误

```typescript
type AIError =
  | { code: 'INVALID_KEY'; message: 'API Key 无效' }
  | { code: 'QUOTA_EXCEEDED'; message: 'API 配额超限' }
  | { code: 'NETWORK_ERROR'; message: '网络连接失败' }
  | { code: 'RATE_LIMIT'; message: '请求过于频繁，请稍后重试' };
```

### 5.3 用户提示系统

- **操作成功**：绿色 Toast 提示，3秒后自动消失
- **操作失败**：红色 Toast 提示，显示具体错误信息
- **加载中**：禁用相关按钮 + 显示 loading spinner

### 5.4 空状态处理

- **无会话数据**：显示引导页（"请先在微信中发起对话"）
- **无聊天记录**：显示空状态插图 + 友好提示
- **AI 摘要加载中**：显示 skeleton 加载效果
- **AI 摘要失败**：显示错误原因 + 重试按钮
